# SlangBase

> 每个行业都有自己的黑话。SlangBase 就是把这些黑话翻译给 LLM 听的小工具。

## 这是啥 🤔

一个团队内部用的 B/S 小工具，用来整理某个领域里的术语和它们之间的关系，最后打包成下游 RAG / Agent 能直接吃的知识。

通用大模型啥都懂一点，但对具体行业的"行话"理解经常翻车——同一个词在不同领域意思完全不一样，专业缩写更是张口就乱猜。与其每次现场跟模型解释半天，不如把团队脑子里的黑话沉淀下来，做成一份结构化的、可以版本化分发的领域字典，让下游系统照着用就完事。

顺便说一句，slang 这个词本身就够西海岸了，所以这名字其实TM合适的 🌴

## 它能做什么 ✨

整体功能就三块：

- 📊 **仪表盘**：看看库里现在有多少东西、最近谁改了啥，以及一个全局搜索框。
- 📚 **术语管理**：核心区，里面有：
  - **术语字典**：录入和维护每一条术语。
  - **知识图谱**：把术语之间的关系（上下位、同义、整体-部分、相关……）连成网，可视化浏览。
  - **发版管理**：随时打个版本快照，导出成下游能用的格式。
- ⚙️ **系统配置**：标签、关系类型、用户、个性化设置之类的杂项。

## 工作流大概长这样 🔁

1. 登录，开整。
2. 把术语录进去，打上标签，配上关系。
3. 觉得攒够了，就发个版，给它一个版本号。
4. 在导出中心把这个版本导出来，扔给下游业务系统去用。
5. 之后再继续改、继续发新版。

## 用给谁 👥

- **维护者**：团队里负责整理领域知识的同事。
- **下游**：自己的各种业务系统——智能问答、文档解析、Agent、报告生成等等，都从这里取知识。

## 暂时不打算做的 🚧

- LLM 辅助自动抽词、自动建关系
- 审核流、复杂权限角色
- 备份恢复、多组织 / 多租户
